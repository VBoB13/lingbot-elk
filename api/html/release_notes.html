OptimusQA - Release Notes

Version 2.0

Release date: June 1, 2023

NEW Features

- Each document uploaded through the /upload endpoint is chunked up and saved with embeddings.

- Each file gets put in its own index for separation of concerns.

- Each file index gets an automatically generated description added onto them.

- For every query sent through /search-gpt will now be decided through which of the current [vendor_id]'s index to look for information from based on this description.

- A parameter strict: Bool can be provided (default: False) to make algorithm act slightly different behind the scenes: strict = True will make the algorithm use LangChain's Agent to try to answer the question. It's slower, but often more accurate. The opposite is true when strict = False; faster, but slightly less accurate.

Changed features

- Previous /upload/[csv|pdf|docx] is now replaced with a single /upload endpoint.

- Uploaded context is now searched for semantically instead of through pure text queries.

Removed features

- Removed specific endpoints for specific filetypes for uploading materials.

- Removed the possibility to search based on text.

- Removed the possibility to search for documents based on timestamps.
